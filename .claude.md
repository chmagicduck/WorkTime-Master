# 工时管理大师 (WorkTime-Master)

## 项目概述

一款帮助职场人精准管理工时、追踪收益的微信小程序。

- **表面**：专业工时管理工具
- **内核**：打工人专属时间账本 & 摸鱼体面版仪表盘
- **Slogan**：你的时间，值得被精准管理。

## 技术栈
- 微信小程序原生开发 (TypeScript + WXML + WXSS)
- TypeScript 严格模式 (`strict: true`, `noImplicitAny`, `strictNullChecks`)
- Skyline 渲染引擎已启用
- 不使用任何 UI 框架，纯原生实现

## 核心业务逻辑

### 状态模型

用户有三种工作状态（存于 `storage.KEYS.CURRENT_STATUS`）：
- `working`：正常上班（工作时段内默认状态）
- `fishing`：摸鱼（仅工作时段内可切换）
- `overtime`：加班（仅下班后可切换；不计算收益，用于展示无偿加班时长）

### 时间 & 配置（存于 Storage）

| Key | 说明 | 默认值 |
|-----|------|--------|
| `MONTHLY_SALARY` | 月薪（税前，元） | 必填 |
| `WORK_START_HOUR/MIN` | 上班时间 | 9:00 |
| `WORK_END_HOUR/MIN` | 下班时间 | 18:30 |
| `SALARY_DAY` | 发薪日（每月几号） | 15 |
| `HOURLY_WAGE` | 时薪（由月薪自动计算，不直接输入） | 自动 |
| `WORK_MODE` | 工作日模式：double_rest / sat_only / sun_only / alternate | double_rest |
| `LUNCH_BREAK_ENABLED` | 午休抵扣开关 | false |
| `LUNCH_START_HOUR/MIN` | 午休开始时间 | 12:00 |
| `LUNCH_END_HOUR/MIN` | 午休结束时间 | 13:00 |
| `ONBOARDING_DONE` | 是否完成引导配置 | false |

详细说明见 `docs/研发文档.md`

### 收益计算

- 摸鱼收益 = `HOURLY_WAGE × (今日摸鱼展示秒 / 3600)`
- 搬砖收益 = `HOURLY_WAGE × (今日搬砖展示秒 / 3600)`（不含加班）
- overtime 段展示「无偿加班时长」，不计入收益
- 工时秒数采用「已完成段累计 + 当前段实时计算」策略，不每秒写 Storage
- 每日 0 点通过 `LAST_RESET_DATE` 对比触发数据重置（在 onShow 中检测）

### 定时器架构

- `Timer` 类以 100ms 间隔运行，内部通过 `eventBus.emit('timer:second')` 每秒触发
- 页面在 `onLoad` 中监听 `timer:second` 事件，每秒批量 `setData` 更新 UI
- 必须在 `onUnload` 中 `timer.stop()` 并 `eventBus.off()` 清理监听

### 页面导航

- 使用 `custom-tab-bar` 组件 + `wx.switchTab` 在四个主页面间切换
- 每个 tab 页的 `onShow` 中调用 `this.getTabBar().setData({ selected: index })` 同步高亮状态

### 兼容性和易错点指南
1、类字段初始化：禁止 ES2022 类字段声明时赋值，必须在 constructor 中初始化。
2、空值合并运算符：禁止使用 ?? 运算符，必须使用 || 或三元表达式替代。
3、系统信息 API：禁止使用 wx.getWindowInfo()，必须使用 wx.getSystemInfoSync() 替代。
4、事件类型定义：缺少特定事件类型（如 TouchEvent），必须使用基础 WechatMiniprogram.Event 或自定义接口。
5、动态属性赋值：TypeScript 严格模式下禁止直接给 Page 对象添加属性，必须使用 (this as any) 或扩展接口。
6、UI 库依赖：禁止引入依赖 React/DOM 的库，图标必须使用 Unicode/Emoji/iconfont 方案。
7、Storage API：禁止混用同步与异步 Storage API，防止数据读写不一致。
8、setData 频率：禁止高频调用 setData（间隔需 >200ms），必须合并数据批量更新。
9、生命周期管理：区分 Page 与 Component 生命周期差异，必须在 onUnload/detached 中清理定时器与监听器。
10、npm 配置：project.config.json 必须设置 packNpmManually: false 以启用自动构建。
11、时区处理：时间计算需显式处理本地时区与 UTC 差异，避免直接隐式依赖系统时区。
12、Skyline rendererOptions：禁止在 app.json rendererOptions.skyline 中使用 tagNameStyleIsolation 字段（Skyline 1.4.7 不支持，会报 invalid 错误）。
13、Skyline 不支持 vh/vw 单位：页面高度必须通过 wx.getSystemInfoSync().windowHeight 获取后以 style="height:{{pageHeight}}px" 注入，禁止使用 min-height:100vh 等视口单位。
